# Full-Site Adversarial Audit (3-Role Challenge)

Date: 2026-03-05

Method:
- Role 1 (Conversion Lead): proposes direct growth/performance moves.
- Role 2 (UX/Frontend Engineer): challenges usability, accessibility, and implementation risk.
- Role 3 (SEO/Ops Lead): challenges discoverability, legal/compliance, and operational reliability.
- Final Verdict: only actions surviving both challenges.

## Global findings (all pages)

### Critical
1. Metadata coverage gap: most pages have no page-level metadata/canonical.
- R1: Add metadata to all indexable pages first.
- R2 challenge: Ensure title/description are unique and route-accurate, not boilerplate.
- R3 challenge: Canonical URL source must be centralized to avoid mixed domains.
- Verdict: Implement metadata baseline for all pages + central `SITE_URL` utility enforcement.

2. Tracking flow is simulated, not real (`/tracking`).
- R1: Keep UI but wire real backend status.
- R2 challenge: Expose clear "demo" state until backend is live to avoid false trust.
- R3 challenge: Fake logistics status on public page is reputationally risky.
- Verdict: Either real API integration or explicit demo badge + noindex until real.

3. Support IVR has likely incorrect contact values (Delhi style numbers and legacy mailbox labels).
- R1: Replace with verified support channels.
- R2 challenge: Ensure tap-to-call and tap-to-mail actions are correct on mobile.
- R3 challenge: Wrong contact info creates lost leads and support failures.
- Verdict: Replace contact data from single config source and add test assertions.

4. `/solutions/[category]` is too thin vs the rest of the site.
- R1: Expand each solution category to a full conversion page.
- R2 challenge: Keep consistent visual system with products pages.
- R3 challenge: Thin pages weaken SEO and internal-link value.
- Verdict: Build a full solution template with proof + CTA + related products.

5. Sitemap coverage is incomplete for many public pages.
- R1: Add all indexable routes.
- R2 challenge: Exclude redirects/utility pages intentionally.
- R3 challenge: Align sitemap with robots and canonical rules.
- Verdict: Expand sitemap for indexable pages and explicitly exclude utility/admin/redirect routes.

### High
1. Mixed copy style (US/UK and terminology drift: `organisations/organizations`, `2D configurator` vs module configurator naming).
2. Legacy fallback domain appears in multiple SEO helpers (`ourwebsitecopy2026-02-21.vercel.app`).
3. Social page uses mock feed content and emoji-heavy captions.
4. Downloads page is request-driven only, but messaging implies direct downloadable assets.
5. Duplicate legacy layout stack (`components/layout/*`) appears unused and increases maintenance risk.
