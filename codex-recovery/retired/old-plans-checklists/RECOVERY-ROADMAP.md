# Recovery Roadmap

## Purpose

This is the short phase map for recovery.

Use:
- `WORLD-CLASS-PLAN.md` for the full standard
- `NEXT-PLAN.md` for the current execution block
- `RECOVERY-CHECKLIST.md` for live status

## Locked Direction

- Supabase stays primary
- Nhost stays fallback or mirror
- archive before delete
- stale plans must not override current verified state

## Completed Foundation

- recovery control system exists
- root cleanup and duplicate-asset archival were completed
- proven-dead source review and `unused/` setup were completed
- route truth and duplicate-family protection were established
- build stability was preserved through the first recovery waves

## Active Recovery Frame

1. platform truth and fallback safety
2. catalog correctness and identity
3. ownership and salvage classification
4. visual-system and homepage quality
5. discovery, PDP, configurator, and trust improvements
6. QA, accessibility, and release hardening

## Rule

Every roadmap step must be translated into a bounded, checklist-driven `NEXT-PLAN.md` block before implementation.
