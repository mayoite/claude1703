# Master Objective

Recover the repo and site without making the app more fragile.

## Mission

- preserve important files before simplifying
- reduce structural ambiguity
- keep the live app stable
- keep recovery state readable without chat history

## Core Rules

- preserve before simplify
- evidence before movement
- archive before delete
- public behavior beats tidiness
- use small reversible steps

## In Scope

- recovery-state docs
- safe root and archive cleanup
- proven-dead source review
- `unused/` as a salvage pool
- verification after each movement wave

## Out Of Scope

- casual deletion of source
- unsafe route moves
- broad redesign done without verification
- collapsing uncertain duplicate systems by guesswork

## Folder Rules

- `codex-recovery/` is the recovery control center
- `archive/` preserves history and duplicates
- `unused/` holds only proven-dead source or reviewed salvage candidates

## Working Directive

Make the repository structurally sane again without throwing away anything that might still matter.
